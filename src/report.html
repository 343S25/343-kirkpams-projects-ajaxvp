Report

  1. The project is available here on GitHub.
  2. The website is available here on w3stu.
  3. The website does not use media queries to create large and small layouts. These layouts are automatically implemented via the grid system available through Bootstrap.
  4. The website has four distinct pages: home, about, history, and settings. The content of the "Add Expense" modal is sufficient to support a fifth page, but it was made into a modal for ease of usage.
  5. The website has a header that is visible on every page.
  6. The website has navigation links in its header.
  7. The website has graphical material through the chart.js library. This material uses the <canvas> element, which does not support the 'alt' attribute. Instead, these elements have their 'role' set to 'img' and have an 'aria-label' attribute describing their content. In addition, two images are used on the 'About' page with adequate 'alt' descriptions appended.
  8. The website passes accessibility and validation checks.
  9. The website uses two font faces: IBM Plex Serif (regular content) and IBM Plex Mono (graphs)
  10. The website has a modal called "Add Expense," which functions as a form and uses form elements.
  11. The website links to my LinkedIn on the "About" page.
  12. The website renders content generated by the chart.js library. On the settings page, you can view a selection for different currency types to select as the default. This functionality is not yet implemented, but a fetch to exchange-api called is made to grab the list of currencies and then another is made to the same API to grab exchange rates to be used internally.
